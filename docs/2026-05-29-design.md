# aiw Design

## Entry Model

The intended entrypoint is cmux-first:

```text
cmux plus / command palette
  -> AI Worktree action
    -> aiw cmux-new --agent <agent>
      -> dependency gate
      -> Worktrunk creates/switches worktree
      -> aiw layout --agent <agent>
      -> cmux opens four-pane workspace
```

`aiw` may also be called directly from a terminal, but cmux is the bridge that makes the workflow available under `~/Code`.

## Non-goals

- No dev server startup by default.
- No project-local personal workflow files by default.
- No replacement for git, editor, diff viewer, file browser, or agent CLIs.
- No automatic `git init` in non-Git directories.

## Four-pane Layout

```text
+----------------------+----------------------+
| Files                | Agent                |
| aiw files            | codex/claude/etc.    |
+----------------------+----------------------+
| Git                  | Diff                 |
| aiw git              | aiw diff --watch     |
+----------------------+----------------------+
```
